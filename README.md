# 文献智读

《文献智读》是一个面向大学生课程写作场景的学习辅助 Web 工具。当前版本基于 `Python + Streamlit` 实现“上传单篇论文 PDF -> 提取关键信息 -> 生成课程写作辅助内容 -> 导出结果”的闭环，重点服务于课程汇报、课程论文前期整理和文献综述基础框架搭建。

## 当前已实现功能

- 首页采用卡片式引导和演示预览，主入口延迟加载解析依赖，减少首屏等待
- 全局背景提供多彩泡泡与碰撞波纹动效，并兼容移动端和减少动态效果设置
- 上传单篇 PDF，先做文本质量预检，再从多种提取策略中选择最可靠结果
- 上传页按阶段展示 PDF 读取、文本提取、元数据识别、结构化提取、AI 解读和写作提纲生成进度
- 识别论文标题、作者、关键词；规则识别不足时可用 AI 辅助补全标题、作者、关键词和摘要
- 主摘要优先显示中文摘要；无中文摘要时回退英文摘要
- 提取研究问题、研究方法、核心结论
- 生成课程写作向的 AI 解读内容：
  - 通俗摘要
  - 研究方法说明
  - 创新点分析
  - 不足分析
- 生成三类写作输出：
  - 课程汇报提纲
  - 课程论文提纲
  - 文献综述基础框架
- 导出 `.md` / `.txt` 解析结果
- 在模型服务不可用时自动回退到本地保守生成，保证主流程可运行
- 结果页在部分成功时保留解析提示，便于复核 AI 补充字段、低置信度关键词和作者识别结果

## 本次迭代新增内容

- 首页和上传页视觉同步升级：卡片入口、文件状态卡片、阶段进度条和错误提示保持同一套界面语言
- 背景泡泡数量增加，并加入碰撞形变与扩散波纹，让背景动效更接近真实碰撞反馈
- 上传流程改为即时显示文件选择状态和解析入口，避免首页卡片里的“上传论文，开始解析”相关入口长时间不出现
- 元数据识别阶段现在会等 AI 补充识别结束后再标记完成或部分完成，避免进度提示提前显示“已完成”
- AI 元数据补全成功时保留调试信息，便于排查实际走了 Responses 还是 Chat Completions 路径
- 结果页左侧恢复“解析提示”区域，部分成功时能直接看到需要人工复核的字段
- 接通作者字段，并统一进入标准结果、页面展示和导出
- 修正摘要语言选择逻辑，避免英文摘要误顶替主摘要
- 收紧关键词去重与数量控制，并在低置信度场景下增加 LLM 关键词兜底
- 新增课程写作辅助层，基于摘要、结构化字段和正文预览生成更适合大学生使用的解读和提纲
- 结果页升级为三段式阅读布局：
  - 左侧：基础信息卡片
  - 中间：摘要、结构化内容、AI 解读
  - 右侧：写作输出区
- 导出文件同步加入新增字段和三类提纲
- 新增 `.env` 文件支持，无需每次手动设置环境变量
- 新增同会话内存缓存，重复上传同一文件时直接复用解析结果，跳过 LLM 调用
- 新增文件大小校验，超过 20 MB 时给出明确提示
- 调试日志统一改用 `logging` 模块，不再输出到标准输出
- 新增 PDF 多策略预检：记录提取策略、质量分、扫描版/低文本密度/乱码诊断，并把诊断信息带入调试结果
- 新增真实论文 PDF corpus 清单和可跳过回归测试，便于持续扩充不同版式样本

## 运行方式

1. 安装依赖

```bash
pip install -r requirements.txt
```

2. 可选：配置中转站环境变量

如果你希望启用结构化重写、关键词兜底和课程写作辅助的 LLM 增强层，复制 `.env.example` 为 `.env` 并填入实际值：

```bash
cp .env.example .env
```

```ini
RELAY_API_KEY=你的中转站密钥
RELAY_BASE_URL=你的中转站兼容地址
RELAY_MODEL=你的模型名
```

未配置时，系统会自动回退为本地规则和保守生成结果，不影响主流程使用。

3. 启动应用

```bash
streamlit run app.py
```

4. 打开浏览器中的本地地址，依次完成：

- 首页进入
- 上传 PDF
- 查看文件状态并开始解析
- 查看结果页与导出结果

## 结果页说明

结果页采用三段式布局，适合课堂展示截图：

- 左侧：展示文件名、摘要语言、作者、关键词和解析提示
- 中间：展示主摘要、通俗摘要、研究方法说明、研究问题/方法/结论，以及创新点和不足分析
- 右侧：展示课程汇报提纲、课程论文提纲和文献综述基础框架

## 当前限制

- 当前版本仍以单篇 PDF 为主，文件大小不超过 20 MB
- 扫描版 PDF 会被识别为低文本密度或图片型文档；未配置可用 OCR 时会给出明确诊断而不是空白结果
- 作者、关键词和结构化字段仍受原始版式质量影响，建议结合原文核对
- 内存缓存仅在同一会话内有效，关闭浏览器后缓存清空
- 多文献对比、批量处理和数据库持久化暂未纳入当前版本

## 测试

当前测试位于 `tests/` 目录，覆盖：

- 应用启动时不提前加载解析流水线
- 首页、结果页和背景动效的关键前端结构
- 关键词显式标记区提取
- 关键词去重与数量控制
- 中文摘要优先于英文摘要
- PDF 预检策略、质量分和短文本诊断
- 可选真实论文 PDF corpus 回归；本地未下载 PDF 时自动跳过
- 作者字段进入标准结果
- AI 元数据补全保留调试信息，元数据识别进度不会提前结束
- 课程写作辅助在信息不足时的稳定 fallback

如果环境未安装 `pytest`，可以直接运行：

```bash
python -m unittest discover -s tests -v
```
